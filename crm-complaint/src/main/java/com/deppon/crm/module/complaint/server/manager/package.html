    RULE-A4 重复投诉码：
    1. 点击列表中投诉人，则显示投诉详细信息，首次点击列表中处理编号，则当前上报投诉根据选择处理编号自动生成重复投诉码，生成规则为：TS+选择的处理编号日期后的流水号，如点击处理编号为201212120007的处理编号，则生成重复投诉码为：TS0007，表示当前上报投诉与列表点选的处理编号投诉为重复投诉，实现两者的系统绑定，上报提交后列表显示重复投诉码；
    2. 点击列表中其他处理编号时，点选处理编号已为重复投诉，则自动读取其重复投诉码，实现三者的绑定；点选处理编号不为重复投诉，则根据生成规则，自动更换重复投诉码，实现两者的绑定；
    3. 重复投诉码可以手动删除；
    4. 点击列表重复投诉码，则弹出新窗口显示所有关联的上报投诉；
    5. 上报类型为异常的投诉，不能生成重复投诉码；
    6. 上报类型为投诉的投诉，不能选择同单号上报类型为异常的投诉生成重复投诉码，也不能选择不同单号的投诉生成重复投诉码；

    1. 任务部门为责任反馈人时，OA系统收到投诉（异常）通知，投诉通知流入责任人反馈人部门的部门投诉管理界面，可录入回访；
