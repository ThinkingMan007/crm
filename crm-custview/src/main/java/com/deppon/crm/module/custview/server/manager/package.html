客户360视图 1. 点击合同信息，合同列表显示合同基本信息，包含审核编号、付款方式、优惠类型、生效日期、失效日期、月结额度、协议联系人、联系方式；选择其中一条合同信息，下方显示合同详细信息以及创建更新信息，数据来源于CRM系统。
2. 点击维护信息，显示计划/日程详情和维护记录，数据来源于CRM系统。
3. 点击订单信息，查看客户的订单号、运单号、订单来源、托运人、货物名称、件数、重量、体积、订单状态、始发网点、受理部门。客户只有通过网上营业厅或者营业部绑定联系人编码，用户ID绑定联系人编码后才可以查询到订单信息。
客户360视图设计
提供查看某一个会员包括客户基础信息、联系人信息、财务信息、合同信息、维护信息、订单信息、积分信息、投诉理赔信息以及运营决策分析等九个模块的360度视图。
RULE-A1根据会员客户编码，查询出包括客户基础信息、联系人信息、账户信息、合同信息、维护信息、订单信息、积分信息、投诉理赔信息以及运营分析等九个模块的360度视图。
RULE-A2 1. 会员360°视图综合信息页中，显示客户主要信息、联系人信息、投诉、理赔、维护、合同信息。“最近一次交易日期”的数据为每天凌晨一统计生成报表，直接读取报表信息，生成折线图。
2. 综合信息显示客户主要信息：客户编码、客户名称、合同类型、税务登记号、客户等级、客户总积分、业务类型、客户行业、联系人基本信息。
3. 联系人信息全部显示，超过3条联系人时，呈现下拉框形式
4. 历史投诉统计、历史理赔统计数据来源于报表；最近一次投诉明细、最近一次理赔明细数据来源于CRM投诉信息、理赔信息。
5. 维护信息读取CRM系统中的客户维护记录，综合信息中显示最近一条维护信息，合同信息读取CRM系统数据，显示最近一条有效合同信息。
6. 点击打印此页，打印出综合信息页
RULE-A3 1、 点击客户基本信息，可以查看客户的基础信息和归属信息，基本信息来源于CRM系统，归属信息记录创建会员的时间、创建人、创建部门，维护会员信息的最后更新人、更新部门、更新时间。
2、 点击联系人信息，联系人列表显示联系人的基本信息，包含联系人编码、联系人姓名、联系人类型、性别、职务、手机号码、固定电话、是否主联系人；选中某条联系人，下方显示此联系人的详细信息以及创建更新信息，数据来源于CRM系统。
3、 点击账户信息，账户列表显示账户基本信息，包含开户行、支行名称、银行账号、开户姓名、账号与客户关系、联系方式；选择其中一条账户信息，下方显示账户详细信息以及创建更新信息，数据来源于CRM系统。
RULE-A4
RULE-A51. 点击积分信息，可以查看客户的总积分、客户可用积分、是否允许联系人兑换积分、以及各个联系人的积分情况；选中一条联系人，下方显示该联系人积分兑换记录。
2. 点击投诉理赔信息，可以查看投诉理赔详细明细，数据来源于CRM系统。
3. 历史投诉统计列表展示字段：投诉范围、保险理赔、代理问题、非投诉、费用投诉、服务、开单差错、客户催单、客户原因、内部投诉、人品问题、时效、业务差错
4. 最近一次投诉明细列表展示字段：处理编号、运单号、投诉人、联系方式、投诉时间、投诉类型、处理状态
5. 历史理赔统计列表展示字段：出险类型、整件丢失、整票丢失、内物短少、破损、潮湿、污染、冒领、其他
6. 最近一次理赔明细列表展示字段：运单号、出发部门、出险类型、理赔类型、理赔金额、处理状态
7. 综合信息中维护信息列表展示字段：联系人姓名、手机号码、固定电话、维护主题、客户意见、具体维护方式、维护时间
8. 合同信息列表展示字段：审核编号、付款方式、优惠类型、生效日期、失效日期、月结额度、协议联系人、手机号码
9. 计划/日程详情列表展示字段：计划制定人、执行人、名称、手机号码、固定电话、客户意见、具体开发方式、开发时间
10. 维护记录列表展示字段：计划制定人、执行人、联系人姓名、手机号码、固定电话、维护主题、客户意见、具体维护方式、维护时间
11. 订单信息列表展示字段：订单号、运单号、订单来源、托运人、货物名称、件数、重量、体积、订单状态、始发网点、受理部门
12. 积分信息：联系人姓名、可用积分、本期累计积分、累计总积分、冻结积分、最后积分时间、积分失效日期
13. 投诉信息列表展示字段：处理编号、运单号、投诉人、联系方式、投诉时间、投诉类型、处理状态、解决情况、客户满意度
14. 理赔信息列表展示字段：差错编号、运单号、出发部门、出险类型、理赔类型、理赔方式、理赔金额、处理状态
15. 点击运营分析，可以查看客户各个维度的数据，表格最多显示12个月的数据。货量结构、使用产品、保营比、包营比、代收货款、渠道偏好等数据来源于报表。根据每个维度的数据形成柱形图、折线图、饼图、条形图等图形。点击导出按钮，可以导出运营分析报告，报告为EXCEL文档，文档格式固定，可以手工修改。
