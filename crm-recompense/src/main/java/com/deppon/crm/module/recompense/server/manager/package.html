理赔出险上报
RULE-A1 当选择理赔类型为“异常签收理赔”和“丢货理赔”时，出险信息中的“运单号/差错编号”处显示为“运单号”；当选择理赔类型为“未开单理赔”时，出险信息中的“运单号/差错编号”处显示为“差错编号”； 查询运单或差错
RULE-A2
  1. 当选择理赔类型为“异常签收理赔”时，输入运单号 ，点击确定，系统校验是否重复上报出险，若是，则提示“该单号已上报出险！不能重复上报”；否则，系统判断该运单号派送出库时是否为异常签收（出库时选择“异常-货物破损”），若是，则可继续，并自动带出运单相关信息（包装、运输类型、保价人（即发货联系人）、联系方式、收货部门、目的站、发货日期、货物名称、件/重/体、保价金额、始发站）；否则，系统判断该运单号是否有OA异常签收的差错记录（偏线/空运/整车/拒收），若是，则可以继续，并自动带出运单相关信息（包装、运输类型、保价人（即发货联系人）、联系方式、收货部门、目的站、发货日期、货物名称、件/重/体、保价金额、始发站）。反之，系统提示“该单号尚未异常签收，不允许上报出险”。
  2. 当选择理赔类型为“丢货类理赔”时，输入运单号 ，点击确定，系统校验是否重复上报出险，若是，则提示“该单号已上报出险！不能重复上报”；否则，系统判断该运单号是否有OA丢货类（有单无货（丢货））的差错记录，若是，则可以继续，并自动带出运单相关信息（包装、运输类型、保价人（即发货联系人）、联系方式、收货部门、目的站、发货日期、货物名称、件/重/体、保价金额、始发站）。反之，系统提示“该单号尚未上报丢货类差错，不允许上报出险”。
  3. 当选择理赔类型为“未开单类理赔”时，输入OA未开单类差错的差错编号 ，点击确定，系统校验是否重复上报出险，若是，则提示“该差错编号已上报出险！不能重复上报”；否则，系统判断该差错编号是否有OA未开单类差错的记录，若是，则可以继续，并自动带出OA差错上报的相关信息（包装、保价人（即发货联系人）、联系方式、收货部门、目的站、发货日期、货物名称、件/重/体、保价金额）。反之，系统提示“该差错编号不是未开单类理赔差错，不允许上报出险”。
  
RULE-A3 出险日期：必须早于或等于报案日期。  
RULE-A4 出险类型：
  1. 理赔类型为异常签收理赔时，下拉列表有：“破损、潮湿、污染、内物短少、冒领及其他”。
  2. 理赔类型为丢货理赔时，下拉列表有：“整票丢失、整件丢失”。
  3. 理赔类型为未开单理赔时，下拉列表有：“破损、潮湿、污染、内物短少、整件丢失、整票丢失及其他”。
  
RULE-A5 所属区域从公司组织结构中带出，系统默认为报案部门所在大区；可进行修改。  
RULE-A6

理赔方式：

  1. 快速理赔：索赔金额必须限制在1000元（含）以下 。上报后，走系统自动处理流程。当理赔类型选择为“未开单类理赔”时，“快速理赔”选项为灰，不可选择。
  2. 正常理赔时，不限制索赔金额。上报后，走理赔专员处理流程。
  3. 在线理赔，默认为灰，不可选择。
  
RULE-A7 索赔方：发货方或收货方。若发/收货方为会员，则自动带出客户编码、客户名称及客户等级。  
RULE-A8 出险描述：必填。可以增加、删除记录。其中，类型与出险类型一致（下拉选项），出险描述每条不少于30个字。  
RULE-A9 货物托运清单及货物索赔清单：默认为灰不可编辑。当理赔方式选择为正常理赔时，可编辑，新增及删除记录。且正常理赔时为必填项。

RULE-A13

对于快速理赔，理赔上报完成后系统自动处理。

  1. 处理意见中：根据上报理赔的金额，自动按如下划分：出发部门所属大区70%，报案部门所属大区30%；如果出发部门和报案部门在同一个大区，则只显示一条记录，且费用为实际上报赔偿金额。

RULE-A10
  1. 处理意见:
    1. 入部门费用：可选择大区，也可以选择营业部；费用为整数，且费用之和等于实际赔偿金额。实际金额小于等于正常理赔与多赔金额之和。
    2. 责任大区：默认为报案部门所在大区。也可进行修改，从系统中选择大区。
    3. 系统消息/短信通知对象：可从通讯录中选择（工号、姓名和部门）；录入相应信息后，系统将信息提示相应人员（待办事宜）。
  2. 追偿信息：当有偏线、空运、中转货物出险时，可根据实际需要填写追偿日期/部门、暂扣金额、考核奖励。当赔偿金额追回后，相关部门对赔回金额、赔回日期、追回暂扣款进行录入。

1. 处理保存时，无须校验填写完整即可保存。

RULE-A12
  1. 在线理赔：理赔专员确认理赔金额及系统自动划分的责任部门，点击“提交审批”，则系统自动生成OA付款工作流。此时，理赔处理状态为“审批完成”，对应于网上营业厅状态为“付款中”。
  2. 快速理赔：对于修改后的快速理赔单，提交审批后，处理状态更新为“审批完成”，无需走审批流程。
  3. 正常理赔：只有营业部同意初步处理理赔金额（确认金额状态）或多赔工作流审批完成（多赔已审批）后，理赔专员才可提交审批；提交审批，则系统生成OA理赔工作流。此时，处理状态更改为：审批中。
    A. 提交时，必填如下信息“正常理赔金额、实际金额、处理意见（入部门费用、责任大区）”；理赔方式为正常理赔，仍需必填系统消息/手机短信通知。（备注：相关规则参见初步处理保存RULE-A10）
    B. 提交审批后，系统根据金额流至相应的领导审批流程（通过报案部门对应到相应领导）：赔偿金额在1000元（含）以下，系统自动审批完成；1000元到5000元（含）区长审批；5000元到10000元（含）大区总审批；10000元以上流至大区总、事业部总裁、营运副总裁及总裁审批。 C.实际理赔金额在1000元以下时，提交审批后，状态直接更改为“审批完成”。
    D. 如果该理赔单有后续创建的多赔单，则：
    D1. 提交时填写的实际理赔金额必须要小于等于多赔单合计金额。
    D2 提交审批时，系统提示信息“该理赔单有对应的多赔单，请核实实际理赔金额”。
    D3. 提交审批后，处理状态更新为“审批完成”，无需走审批流程。

理赔付款
RULE-A1 通过相应查询条件，选中运单后，进入该单的理赔单查询界面。  
RULE-A2 理赔审批完成后，由营业部收银员进行付款操作。只有部门收银员才可进行付款。 创建理赔付款申请/
保存付款申请/
提交付款申请
RULE-A3 工号、理赔方式、理赔类型、运单号/差错编号、理赔金额、客户编码、客户等级、保价人，系统根据理赔单信息自动填充。  
RULE-A4 冲账信息中冲应收账款及冲到付款为可选项。当选中冲应收账款后，对应的冲应收账款金额为可编辑；当选中冲到付款后，对应的冲到付款金额为可编辑。冲应收账款金额与冲到付款金额之和不能为大于理赔金额；冲应收账款金额不能大于理赔金额；冲到付款金额不能大于理赔金额。  
RULE-A5 付款金额为灰不能编辑。且付款金额=理赔金额-冲应收账款金额-冲到付款金额。  
RULE-A6 付款方式及所属财务部默认为空。  
RULE-A7 当付款金额为0时，汇款方式默认为“现金”，不需要填写收款人的账户信息。点击“提交付款”，不需要对接费控系统生成OA工作流，且处理状态更新为“已付款”。  
RULE-A8 当付款金额大于0时，需要选择汇款方式  
RULE-A9
  1. 若汇款方式为“现金”，则系统默认账户信息（开户名、开户银行、开户账号、省份、城市、支行）为收银员的账户信息【通过对接费控系统实现】，若费控系统无法获取到收银员的账户信息，可手工录入；
  2. 若汇款方式为“汇款”，则不需要对接费控系统，直接手工录入客户的账户信息即可。
  3. 开户银行、省份、城市、支行这些字段从银行基础资料获取。（银行基础资料同步费控系统的银行基础资料实现）
点击“提交付款”，系统对接费控系统生成OA付款工作流。此时，处理更改为“付款中”。付款工作流审批完成后，状态更改为已付款。若工作流退回（也包括同意后退回情况），则处理状态更改为付款失败。对于付款失败的理赔单，可再次申请付款。		 付款审批反馈
RULE-A10 打印：对理赔单信息的打印。打印内容参见：DP-CRM-用户功能需求说明书-理赔管理V1.0文档中的UC_CRM_SE_07_03中，4.4.1中理赔打印界面。其中情况描述为：出险情况的类型、数量及描述信息。审批人及审批意见为：理赔工作流的最后审批人及审批意见。  
RULE-A11 反付款：对于理赔金额全部冲应收账款及到付款的情况，允许应收账款小组反付款操作，则处理状态由“已付款”更改为“付款失败”。反付款后，营业部收银员可再次付款申请。 返付款
理赔查询
RULE-A1 进入理赔查询界面，默认列表信息结果为空。 理赔查询
RULE-A2
  1. 上报部门默认为当前组织所在部门。当用户为理赔专员时，可选择上报部门进行查询。
  2. 所属区域默认为上报部门所在区域。当用户为理赔专员时，可选择所属区域查询。当所属区域为空时，查询全公司的数据。
  3. 上报时间与最后审批时间必填其一；当理赔方式选择所有时，可查询时间段内上报部门的所有理赔方式的理赔信息；当理赔类型选择所有时，可查询时间段内上报部门的所有理赔类型的理赔信息；当处理状态选择所有时，可查询时间段内上报部门的所有状态的理赔信息。
  4. 按客户编码进行查询时，上报部门查询条件失效。可直接查询出时间内本客户的所有理赔信息。
  5. 按上报编号查询时，其他所有查询条件失效。
  
RULE-A3

选中运单号/差错编号，可查看理赔单详细信息。

A、理赔单查询信息包括：出险信息、理赔信息、出险描述、货物托运清单、理赔清单、附件、处理意见、追偿信息、工作流与跟进、差错信息、理赔历史记录、多赔记录。

B、按钮显示规则：

  1. 当登录角色为“营业部经理”时，显示“跟进、打印、确认金额、创建多赔、修改、删除”按钮。
  2. 当登录角色为“理赔专员”时，显示“跟进”按钮。
  3. 当登录角色为“收银员”时，显示“打印、付款申请、跟进”按钮。
  4. 当登录角色为“营业员”中时，显示“跟进”按钮。
  5. 当登录角色为“应收账款小组”时，显示“反付款”按钮。
  6. 相关按钮规则限制同以上其他用例描述。
  
RULE-A4 进入多赔查询界面，默认列表信息结果为空。  
RULE-A5
  1. 创建部门为能为空，默认为当前组织所在部门。当用户为理赔专员时，可选择其他部门组织进行查询。
  2. 创建时间必填；
  
理赔监控
RULE-A1 打开理赔监控界面，默认显示所有未处理完成（审批完成状态之前）的钻石会员的理赔单情况。 理赔监控
RULE-A2
  1. 上报时间必填；
  2. 当客户等级选择所有时，可查询时间段内所有客户等级的所有未处理完成的理赔信息。
  3. 按客户编码进行查询时，不需要关联客户编码及所属大区。可直接查询出时间内本客户的所有未处理完成的理赔信息。
  4. 所属大区，默认为空，此时查询结果为公司所有理赔单。
  
RULE-A3
  1. 状态持续时间为：当前时间与当前处理状态时间的时间差。
  2. 通过运单号或差错编号，可链接到理赔单查看界面，对理赔单详细信息查询。
  
RULE-A4 理赔单查询信息包括：出险信息、理赔信息、出险描述、货物托运清单、理赔清单、附件、处理意见、追偿信息、工作流与跟进、差错信息、理赔历史记录、多赔记录。  
RULE-A5 催办后，对应大区的理赔专员，系统信息提醒催办信息（通过待办事宜实现）。且催单次数加1.  
在线理赔提交
RULE-A1
  1. 点击“同意“按钮，可进入下一步。
  2. 点击”拒绝“，则退出在理赔理赔申请。
 在线理赔提交
RULE-A2

获取验证码的规则：

  1. 点击获取验证码，系统自动将验证码通过短信的形式发送到用户的手机上，有手机号，优先发送短信验证码，不需再发送语音短信验证码，无手机号，则以语音的形式发送给用户的电话（限制只可听两次），短信内容模板：您好，这里是德邦物流理赔中心，单号XXXXXXXX的在线理赔验证码：XXXX，若手机号与电话号码不正确，则提示“联系方式不正确，不能申请在先理赔，请与部门联系线下理赔”；
  2. 输入验证码，系统判断验证码是否正确，若不正确，则提示“验证码输入错误，请重新输入验证码”；若验证码超时，则提示重新获取验证码；
  
RULE-A3 索赔金额：0<索赔金额<=1000，且索赔金额<=保价；  
RULE-A4 身份证号：限制位数，只能录入15位或18位；  
RULE-A5

确认所有信息无误，同意理赔协议时，

  1. 系统校验该单号有无派送异常出库（派送出库时，选择签收类型为“异常-货物破损”），若有，则直接进入下一步；否则，校验该单号有无上报丢货差错，若已上报丢货差错则直接生成理赔申请单，否则，系统生成丢货差错申请单及理赔申请单。
  2. 系统自动生成短信给出发部门/到达部门经理（上报时选择发货方则短信通知出发部门经理，上报时选择收货方则短信通知到达部门经理），短信内容模板：您好，单号XXXXX的客户提出在线理赔，请在24小时内处理完毕，谢谢！
  
RULE-A6 当固定客户选择理赔方为发货方时，需将运单中发货方客户编码填入理赔单客户编码；如果为收货方，则根据用户信息找收货客户的客户编码，填入理赔单客户编码；  
RULE-A7 若部门经理未对该单进行审核（受理或拒绝），客户可对在线申请的理赔相关信息进行修改。修改规则同在线理赔提交。

2）填写出险类型、出险数量、所属区域、出险日期及出险描述信息。（规则同FS_CRM_LP_01）
